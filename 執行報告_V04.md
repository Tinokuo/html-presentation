## 背景
本輪接手時，`index.html` 已是 Reveal.js 版互動 deck（10 頁），且 `images/` 內的 AI 配圖（背景 6 張、圖標 5 個、`icon_sheet.png` 總表）皆已由 Draw 技能（gpt-image-2）於前一輪產出。但這些檔案**全數未提交**：`main`／`origin/main`（= 線上 GitHub Pages）仍是更早的自製單檔 deck。需求為「補圖強化 + 排除異常 + 掛上 GitHub」。

## 發現的異常
逐張檢視圖標後，發現 `images/icon_share.png` **裁切殘留**：左緣帶到相鄰「書本」圖標的右側碎片。
- 根因：前一輪以「sheet 寬度等分 5 欄」裁切，但 `icon_sheet.png`（1536×1024）內 5 個圖標並非等距置中，share（第 3 個，x≈681–880）落在等分欄邊界，把鄰圖右緣一起切入。

## 做了什麼
1. **內容感知重切**：以背景色（深藍 RGB≈1,9,21）距離建遮罩，偵測 5 個圖標各自的 bounding box，逐一「置中 + 25% 等比邊距」裁成 256×256，避免鄰圖殘留。
2. 重切 5 圖（density / read / share / interact / joy），再以 `remove_bg.py`（亮度閾值去背）統一邊緣風格。
3. **驗證**（本機 http.server:8123 + Reveal API / DOM eval）：
   - 10 頁全在；封面背景 `cover.png` 已套用並載入。
   - 5 個 `<img>` 圖標 `naturalWidth=256`、全載入；`icon_share` 已無殘留（橘節點接兩青圈）。
   - 伺服器日誌：6 背景 + 5 圖標皆回應 200/304。
   - 註：`preview_screenshot` 因 Firebase `onSnapshot` 長連線使頁面不進入 network-idle 而逾時，屬工具限制，非 deck 缺陷。
