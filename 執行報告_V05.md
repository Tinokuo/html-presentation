# 執行報告 V05 — 文字雲 Firebase 遷移（送出修復）

- 時間：2026-06-24
- 交付物：`index.html` 第2頁即時文字雲
- 問題：文字雲「送出按鈕無反應」
- 模式：正式執行

## 根因
- 送出失敗真因＝ Firestore 寫入 `permission-denied`。
- 簡報原本內建的 `teacherstudy-109ef` 是「工具的共用範例 Firebase」，tino 無管理權限 → 規則改不了 → 一直被擋（文字雲其實從未送出成功過）。

## 解法（改接 tino 自己的 Firebase）
1. 程式端先加「送出失敗顯示紅字原因」（不再靜默無反應）。
2. 本機安裝 `firebase-tools` 14.27.0（npm global）。
3. `firebase login` 完成（tinokuo77@gmail.com）。
4. 發現 tino 已有專案 **tino-422c8（Tino）**，內含現成 web app「html簡報」。
5. 取得該 app 的 firebaseConfig，替換 `index.html`（teacherstudy-109ef → tino-422c8）。
6. tino-422c8 原本未建 Firestore → 透過 Chrome 擴充（Claude in Chrome）代為操作 Firebase 控制台：
   建立資料庫 → Standard 版 → 位置 nam5 → **測試模式**（30 天開放讀寫）→ 建立。
7. 規則生效後實測寫入：**write OK**（寫一筆 + 自動刪除，無髒資料）。
